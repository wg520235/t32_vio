# T32 VIO 项目 - 系统约束分析

## 硬件平台：君正 T32

| 资源 | 规格 | 评估 |
|------|------|------|
| NPU | 1 TOPS | 可用，但需量化/轻量化模型 |
| CPU | 1.2 GHz MIPS | 紧张，必须极致优化 |
| 内存 | 64 MB | **极度紧张**，算法设计的硬约束 |
| OS | Linux | 标准POSIX，可用多线程 |

## 传感器

| 参数 | 规格 | 影响 |
|------|------|------|
| 分辨率 | 320x240 | 低分辨率，特征点稀疏 |
| 帧率 | 180 fps | 高帧率，时间分辨率好，但算力压力大 |
| 曝光 | 全局 | 无卷帘畸变，有利于运动估计 |
| 格式 | NV12 (YUV420) | 可直接用Y通道做灰度处理 |
| 色彩 | 纯黑白 | 无颜色信息，只能用灰度特征 |

## 关键挑战

### 1. 内存约束 (64MB)
- Linux系统本身占用 ~10-15MB
- 留给算法的内存：~45-50MB
- 必须严格管理：图像缓冲、特征点、地图点、优化状态

### 2. 算力约束
- 1.2GHz MIPS 单核性能有限
- 180fps 输入，实际处理可能只能做到 30-60fps
- 需要帧降采样或选择性处理

### 3. 特征稀疏
- 320x240 分辨率低，特征点数量有限
- 纯灰度，无颜色辅助
- 需要鲁棒的特征提取与跟踪

### 4. 实时性要求
- 无人机控制周期通常 200-400Hz
- VIO输出需要至少 50-100Hz
- 延迟必须 < 20ms

## 设计目标

1. **内存占用 < 40MB**
2. **处理帧率 >= 30Hz** (从180fps中选择)
3. **位置漂移 < 1%** (室内/短距离)
4. **延迟 < 15ms**
5. **支持动态场景**
